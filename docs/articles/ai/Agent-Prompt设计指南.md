# Prompt 写得烂，Agent 就会瞎跑

你的 Agent 一直在调用错误的工具，或者给出的回答完全跑题。你以为是 Agent 的问题，其实是 Prompt 写得烂。

我之前做过一个 Agent，让它"帮我分析代码"，结果它开始删除文件。后来才发现，是 System Prompt 写得不清楚，Agent 以为"分析"包括"清理无用文件"。

这篇文章，我来分享怎么写好 Agent 的 Prompt。

## System Prompt 是什么？

Agent 有两层 Prompt：

**System Prompt**：给 Agent 的"人设"和"规则"，整个对话过程中不变。

```python
system_prompt = """你是一个代码助手，帮助用户编写和调试代码。

你可以：
- 读取文件
- 编写代码
- 运行测试

你不可以：
- 删除文件
- 执行危险命令
- 修改系统配置
"""
```

**User Prompt**：用户的输入，每轮对话可能不同。

```
用户：帮我分析一下 main.py
```

Agent 的行为主要由 System Prompt 决定。

## 好的 System Prompt 结构

我总结了一个模板：

```python
SYSTEM_PROMPT = """
# 角色
你是一个 [具体角色]，专门负责 [具体任务]。

# 能力
你可以：
- [能力1]
- [能力2]
- [能力3]

# 限制
你不可以：
- [限制1]
- [限制2]

# 工作流程
当用户请求时，按以下步骤：
1. [步骤1]
2. [步骤2]
3. [步骤3]

# 输出格式
你的回答应该：
- [格式要求1]
- [格式要求2]

# 示例
用户：[示例输入]
助手：[示例输出]
"""
```

## 常见问题

### 问题一：角色定义太宽泛

❌ 错误：
```
你是一个 AI 助手。
```

Agent 什么都想干，什么都干不好。

✅ 正确：
```
你是一个 Python 代码助手，专门帮助用户编写和调试 Python 代码。
```

范围明确，Agent 知道自己该干什么。

### 问题二：限制不够具体

❌ 错误：
```
不要做危险的操作。
```

什么是"危险"？Agent 可能以为删除临时文件不算危险。

✅ 正确：
```
你不可以：
- 删除任何文件
- 执行 sudo 命令
- 修改 .env 文件
- 访问 /etc 目录
```

明确列出禁止的操作。

### 问题三：没有工作流程

❌ 错误：
```
帮助用户解决问题。
```

Agent 拿到问题就直接回答，不考虑上下文。

✅ 正确：
```
工作流程：
1. 理解用户意图
2. 读取相关文件
3. 分析问题
4. 给出方案
5. 验证结果
```

Agent 会按步骤来，不会跳步。

## 动态 Prompt

有时候需要根据情况调整 Prompt：

```python
def build_system_prompt(mode: str, user_role: str) -> str:
    """根据模式构建不同的 System Prompt"""
    
    base = """
你是一个代码助手。
"""
    
    if mode == "plan":
        base += """
当前是 PLAN 模式，你只能：
- 分析代码
- 提供建议
- 生成计划

你不可以修改任何文件。
"""
    else:
        base += """
当前是 BUILD 模式，你可以修改代码和文件。
"""
    
    if user_role == "junior":
        base += """
用户是初学者，解释要详细，代码要有注释。
"""
    
    return base
```

## 我踩过的坑

**坑一：Prompt 太长**

一开始我想把所有规则都写进去，结果 Prompt 有 2000 多字。LLM 看到后面就忘了前面，效果反而变差。

解决：只写最重要的规则，控制在 500 字以内。

**坑二：指令冲突**

Prompt 里写"快速响应用户"，又写"仔细分析问题"。Agent 不知道该快还是该慢。

解决：明确优先级：

```
当速度和质量冲突时，优先保证质量。
```

**坑三：没有边界情况**

用户输入了奇怪的内容，Agent 不知道怎么办，就开始瞎编。

解决：加上兜底规则：

```
如果用户请求超出你的能力范围，明确告知用户，不要尝试执行。
```

## 下一步行动

1. **检查你的 System Prompt**：有没有明确的角色、能力、限制？
2. **加上工作流程**：让 Agent 按步骤执行，不跳步
3. **加几个示例**：Few-shot 让 Agent 更清楚该怎么回答

Prompt 是 Agent 的"大脑设定"，写好了事半功倍。
